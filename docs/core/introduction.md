### <a name="jit-compiler-and-il"></a><span data-ttu-id="919f4-260">JIT コンパイラと IL</span><span class="sxs-lookup"><span data-stu-id="919f4-260">JIT compiler and IL</span></span>

<span data-ttu-id="919f4-261">C# などの高レベル .NET 言語は、中間言語 (IL) と呼ばれるハードウェアに依存しない命令セットにコンパイルされます。</span><span class="sxs-lookup"><span data-stu-id="919f4-261">Higher-level .NET languages, such as C#, compile down to a hardware-agnostic instruction set, which is called Intermediate Language (IL).</span></span> <span data-ttu-id="919f4-262">アプリを実行すると、JIT コンパイラにより、プロセッサで認識されるマシン コードに IL が変換されます。</span><span class="sxs-lookup"><span data-stu-id="919f4-262">When an app runs, the JIT compiler translates IL to machine code that the processor understands.</span></span> <span data-ttu-id="919f4-263">JIT コンパイルは、コードが実行されるのと同じマシン上で行われます。</span><span class="sxs-lookup"><span data-stu-id="919f4-263">JIT compilation happens on the same machine that the code is going to run on.</span></span>

<span data-ttu-id="919f4-264">JIT コンパイルはアプリケーションの実行中に行われるため、コンパイル時間は実行時間の一部になります。</span><span class="sxs-lookup"><span data-stu-id="919f4-264">Since JIT compilation occurs during execution of the application, the compilation time is part of the run time.</span></span> <span data-ttu-id="919f4-265">したがって、JIT コンパイラを使用して、コードの最適化に要する時間と、結果のコードによって得られる時間の節約のバランスを取る必要があります。</span><span class="sxs-lookup"><span data-stu-id="919f4-265">Therefore, JIT compilers have to balance time spent optimizing code against the savings that the resulting code can produce.</span></span> <span data-ttu-id="919f4-266">しかし、JIT コンパイラによって実際のハードウェアが認識されるため、開発者は異なるプラットフォームに対して異なる実装を提供する必要がなくなります。</span><span class="sxs-lookup"><span data-stu-id="919f4-266">But a JIT compiler knows the actual hardware and can free developers from having to ship different implementations for different platforms.</span></span>

## <a name="advanced-scenarios"></a><span data-ttu-id="919f4-415">高度なシナリオ</span><span class="sxs-lookup"><span data-stu-id="919f4-415">Advanced scenarios</span></span>

<span data-ttu-id="919f4-416">以下のセクションでは、高度なシナリオで役立つ .NET のいくつかの機能について説明します。</span><span class="sxs-lookup"><span data-stu-id="919f4-416">The following sections explain some capabilities of .NET that are useful in advanced scenarios.</span></span>

### <a name="native-interop"></a><span data-ttu-id="919f4-417">ネイティブ相互運用</span><span class="sxs-lookup"><span data-stu-id="919f4-417">Native interop</span></span>

<span data-ttu-id="919f4-418">どのオペレーティング システムにも、システム サービスを提供するアプリケーション プログラミング インターフェイス (API) が含まれています。</span><span class="sxs-lookup"><span data-stu-id="919f4-418">Every operating system includes an application programming interface (API) that provides system services.</span></span> <span data-ttu-id="919f4-419">.NET には、その API を呼び出すためのいくつかの方法が用意されています。</span><span class="sxs-lookup"><span data-stu-id="919f4-419">.NET provides several ways to call those APIs.</span></span>

<span data-ttu-id="919f4-420">ネイティブ API と相互運用する主な方法は、"プラットフォーム呼び出し" (略して P/Invoke) を使用するものです。</span><span class="sxs-lookup"><span data-stu-id="919f4-420">The main way to interoperate with native APIs is via "platform invoke" or P/Invoke for short.</span></span> <span data-ttu-id="919f4-421">P/Invoke は、Linux および Windows プラットフォーム全体でサポートされています。</span><span class="sxs-lookup"><span data-stu-id="919f4-421">P/Invoke is supported across Linux and Windows platforms.</span></span> <span data-ttu-id="919f4-422">Windows 限定で相互運用を行う方法は "COM 相互運用" と呼ばれます。これは、マネージド コードで [COM コンポーネント](/cpp/atl/introduction-to-com)を操作する場合です。</span><span class="sxs-lookup"><span data-stu-id="919f4-422">A Windows-only way of interoperating is known as "COM interop," which works with [COM components](/cpp/atl/introduction-to-com) in managed code.</span></span> <span data-ttu-id="919f4-423">これは、P/Invoke インフラストラクチャ上に構築されますが、動作は少し異なります。</span><span class="sxs-lookup"><span data-stu-id="919f4-423">It's built on top of the P/Invoke infrastructure, but it works in subtly different ways.</span></span>

<span data-ttu-id="919f4-424">詳細については、「[ネイティブ相互運用性](../standard/native-interop/index.md)」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="919f4-424">For more information, see [Native interoperability](../standard/native-interop/index.md).</span></span>

### <a name="unsafe-code"></a><span data-ttu-id="919f4-425">アンセーフ コード</span><span class="sxs-lookup"><span data-stu-id="919f4-425">Unsafe code</span></span>

<span data-ttu-id="919f4-426">言語サポートに応じて、CLR の `unsafe` コードによって、ネイティブ メモリにアクセスしたりポインターの算術演算を実行したりできるようになります。</span><span class="sxs-lookup"><span data-stu-id="919f4-426">Depending on language support, the CLR lets you access native memory and do pointer arithmetic via `unsafe` code.</span></span> <span data-ttu-id="919f4-427">この操作は、特定のアルゴリズムおよびシステム相互運用性のために必要です。</span><span class="sxs-lookup"><span data-stu-id="919f4-427">These operations are needed for certain algorithms and system interoperability.</span></span> <span data-ttu-id="919f4-428">アンセーフ コードの使用は強力ですが、システム API との相互運用を行ったり、最も効率的なアルゴリズムを実装したりする必要がなければ、推奨されません。</span><span class="sxs-lookup"><span data-stu-id="919f4-428">Although powerful, use of unsafe code is discouraged unless it's necessary to interoperate with system APIs or implement the most efficient algorithm.</span></span> <span data-ttu-id="919f4-429">アンセーフ コードは、環境が異なると同じように実行されない可能性があり、さらにガベージ コレクターとタイプ セーフの利点が得られない場合があります。</span><span class="sxs-lookup"><span data-stu-id="919f4-429">Unsafe code may not execute the same way in different environments and also loses the benefits of a garbage collector and type safety.</span></span> <span data-ttu-id="919f4-430">可能な限りアンセーフ コードのみに制限し、そのコードを徹底的にテストすることをお勧めします。</span><span class="sxs-lookup"><span data-stu-id="919f4-430">It's recommended to confine and centralize unsafe code as much as possible and test that code thoroughly.</span></span>

<span data-ttu-id="919f4-431">詳細については、[アンセーフ コードとポインター](../csharp/programming-guide/unsafe-code-pointers/index.md)に関するページを参照してください。</span><span class="sxs-lookup"><span data-stu-id="919f4-431">For more information, see [Unsafe code and pointers](../csharp/programming-guide/unsafe-code-pointers/index.md).</span></span>
